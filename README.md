# ハングル読みルビ

韓国語学習者（日本語話者）向けの「ハングル読み方表示Webアプリ」です。
カメラ撮影・写真アップロード・テキスト直接入力のいずれかでハングルを入力すると、
1文字ごとにローマ字（Revised Romanization ベース、メイン表示）とカタカナ読み（補助表示）を表示します。
文字や単語をタップすると韓国語の音声読み上げ（Web Speech API）も行えます。

ビルド不要の単一ページアプリです（`index.html` を開くだけで動作します）。

## ファイル構成

- `index.html` — アプリ本体（UI・OCR呼び出し・表示ロジック）
- `hangul.js` — ハングル→読み変換ロジック（純粋関数、ブラウザ／Node両対応）
- `test.js` — `hangul.js` のテスト（`node test.js` で実行）
- `README.md` — 本ファイル

## 起動方法

カメラ機能はブラウザの制約上、**HTTPS または localhost からのアクセスでないと動作しません**。
そのため `index.html` をローカルサーバー経由で開いてください。

```sh
cd /Users/suzukitakeo/ides/BambooTuna/lang-reader
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000` を開いてください。

（写真アップロードやテキスト入力のみを使う場合は `index.html` を直接ダブルクリックして開いても動作しますが、
カメラタブは前述の理由により file:// では起動できません。）

## 使い方

1. 画面上部のタブで入力方法を選択します。
   - **カメラ**: 「カメラを起動」→「撮影して読み取る」で撮影した静止画をOCRします。
   - **写真アップロード**: ファイル選択・ドラッグ＆ドロップ・クリップボード貼り付け（Ctrl+V / Cmd+V）に対応しています。
   - **テキスト入力**: 韓国語テキストを直接貼り付けると即座に変換されます（OCR不要）。
2. OCR実行時は認識中の進捗・状態が日本語で表示されます。
3. 認識結果は編集可能なテキストエリアに表示されるので、誤認識を手直しして「この内容で再変換」を押すと読み方表示に反映されます。
4. 読み方は単語ブロックごとに、ハングル1文字ずつ縦に「ハングル／ローマ字（メイン・大きめ）／カタカナ（補助）」の3段で表示されます。
   - 「カタカナ表示」トグルでカタカナ行の表示・非表示を切り替えられます（デフォルトON）。
5. 音声読み上げ（タップ再生）
   - ハングル1文字をタップ → その音節を読み上げます。
   - 単語ブロック先頭の🔊ボタン → その単語全体を読み上げます。
   - 結果エリア上部の「🔊 全文を再生」→ 単語ごとに順番に読み上げ、再生中の単語をハイライトします。
   - 再生中の文字・単語は背景色でハイライト表示されます。

## 注意点

- **カメラはHTTPSまたはlocalhostでのみ動作します。** `file://` で直接開いた場合、`getUserMedia` がブラウザにブロックされるため、上記の `python3 -m http.server` 等でローカルサーバーを立ち上げてアクセスしてください。
- **OCRには初回インターネット接続が必要です。** Tesseract.js本体および韓国語の言語データ（`kor.traineddata`）をCDN（jsdelivr）からダウンロードするため、初回実行時はネットワーク接続が必須です。一度ダウンロードされたデータはブラウザにキャッシュされ、以降はオフラインでも動作しやすくなります。
- OCRの認識精度は画像の解像度・傾き・照明条件に大きく左右されます。認識結果はテキストエリアで手直ししてから再変換してください。
- **音声読み上げはサーバー不要のOS内蔵TTS（Web Speech API `speechSynthesis`）を利用します。** インターネット接続やサーバーサイドの音声合成は使用していません。
  - macOS / iOS には韓国語音声（「Yuna」等）が標準搭載されていることが多く、追加設定なしでそのまま読み上げできます。
  - 韓国語音声が端末に無い場合は「この端末に韓国語音声が見つかりません」という警告が表示されます。その場合はOSの設定から韓国語（Korean）の音声・言語パックを追加してください。
    - macOS: 「システム設定」→「アクセシビリティ」→「読み上げコンテンツ」→「システムの声」で韓国語の声を追加
    - iOS: 「設定」→「アクセシビリティ」→「読み上げコンテンツ」→「声」で韓国語を追加
    - Windows: 「設定」→「時刻と言語」→「音声認識」または「言語と地域」で韓国語の音声を追加
  - 音声の対応状況はブラウザ・OSに依存します（Chrome/Edge/Safari推奨）。

## 実装している発音規則（範囲）

`hangul.js` は最低限の発音規則のみを実装しています。

1. **連音化**: 終声＋次音節の初声ㅇ（無音）→ 終声を次の初声として読む（例: 한국어 → 한/구/거 → ハングゴ）。ただし終声ㅇ（ng）は移動しません。
2. **終声の代表音化**: 発音上の終声をㄱ/ㄴ/ㄷ/ㄹ/ㅁ/ㅂ/ㅇの7代表音に単純化します。二重パッチムも代表的なものに単純化します（例: ㄳ→ㄱ、ㄵ→ㄴ、ㄼ→ㄹ、ㅄ→ㅂ）。
3. **語頭／語中による平音（ㄱ・ㄷ・ㅂ・ㅈ）の清濁**: 単語の先頭では無声（カ・タ・パ・チャ行）、2文字目以降では有声（ガ・ダ・バ・ジャ行）として表記します（例: 한국어の「국」は語中なので「グ」）。

### 未実装（意図的に対象外）

- **鼻音化**（例: 합니다の「ㅂ+ㄴ」→「ㅁ+ㄴ」。そのため「감사합니다」は鼻音化前の読みで「カムサハプニダ」と表示されます）
- **流音化**（例: 신라→シルラ）
- **激音化**（例: 좋다→チョタ）
- 二重パッチムの片方のみが連音するケース（例: 값이→갑씨のようなㅅの移動）。本実装では二重パッチムを代表音1つに単純化してから連音化を行うため、この種の変化は再現しません。

これらの高度な規則は実装しておらず、`hangul.js` 内にもその旨をコメントで明記しています。
